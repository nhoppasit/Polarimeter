Agilent 34401A StatRegSRQ Programming Sample

Development ADE:  C#, Visual Studio 2003

IO Library: VISA-COM w/o formatted IO

Description

The Agilent 34401A StatRegSRQ Programming Sample demonstrates how you can use the multimeter's status registers to determine when a command sequence is completed. For more information see "The SCPI Status Model" in the Agilent 34401A User Guide.

NOTE: Polling will only work with GPIB Interface. It does not work for RS232.

  • user4test
  • Initializing an IO connection to the 34401A and then closing the connection
  • Sequence of Operation:
    • The meter is cleared and set to give an SRQ when its operation is complete.
    • The meter is set for dc, and multiple readings. This will take about 3 seconds for 10 readings.
    • Enable the timer that polls the SRQ every minute.
    • When SRQ is detected, then get the reading from the meter with the routine ReadData.

 To build and run 34401A StatRegSRQ, you must have Visual Studio 2003 and Agilent IO Libraries Suite installed on your PC.

Building the Sample

To build the sample, open the solution file StatRegSRQ.sln in Visual Studio 2003.  From the menu, select "Build | Rebuild Solution".

Running the Sample

From the Visual Studio 2003 menu, select "Debug | Start", of press the F5 key.
